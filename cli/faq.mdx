## How is this different from WP-CLI?

WP-CLI runs on your WordPress server and is the right tool for server-side administration. Respira CLI runs locally, talks to the Respira plugin over HTTPS, and understands page builder content natively.

## Does it require SSH?

No. Respira CLI never touches your server directly.

## Which managed hosts work?

All of them. WP Engine, Kinsta, SiteGround, Cloudways, Pressable, Pantheon, Flywheel, and self-hosted WordPress.

## Is it open source?

Yes. MIT license. [github.com/respira-press/respira-wordpress-cli](https://github.com/respira-press/respira-wordpress-cli).

## What does it cost?

Public reads (`respira read structure`, `respira read design-system`) are free with no account. Writes require a Respira license. See [pricing](https://respira.press/pricing).

## Do i need Claude Code or another agent?

No. The CLI is a standard command line tool that works on its own. Agent integrations are optional.

## Can i use it in CI?

Yes. Set `RESPIRA_API_KEY` and run any command. JSON output is the default when stdout is not a TTY.

## Which builders are supported?

Elementor, Divi 4, Divi 5, Bricks, WPBakery, Beaver Builder, Oxygen, Breakdance, Brizy, Thrive Architect, Flatsome UX Builder, Gutenberg, plus WooCommerce.
